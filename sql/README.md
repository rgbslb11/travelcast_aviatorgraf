# SQL Reference

Run the core schema from the separate TravelCast backend starter kit first, then use these files for this app contract:

- `expected_views.sql` — frontend data contract.
- `live_views.sql` — starter live views.
- `functions_airport_impact.sql` — impact helper stubs.
- `sample_seed_airports.sql` — 10 starter airport seed rows.

The frontend should primarily read `v_airport_status_dashboard` and detailed drill-down views.
